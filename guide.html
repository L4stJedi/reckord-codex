Prerequisites
⚙️
What you need before starting
Accounts, licences, and access
▼
1
Google Workspace account
You need a Google Workspace Business Starter (or higher) subscription. Individual Gmail accounts cannot be used for AppSheet with team access.

Minimum recommended plan: Business Starter — covers Sheets, Drive, and AppSheet Core.
For AppSheet automations (Apps Script triggers): Business Standard or higher.

Admin access is required to add users and configure sharing permissions.
2
AppSheet account
AppSheet is included in Google Workspace. Activate it at appsheet.com using your Workspace Google account. No separate subscription is needed for the initial build — AppSheet Core is included with most Workspace plans.

If you need offline mode or advanced automation, you may need AppSheet Enterprise Standard.
3
User accounts for the team
Create Google Workspace user accounts for all staff who will use the platform. Suggested naming: firstname.lastname@reckord.tv

Create the following Google Groups (used for role-based access):
  • operations@reckord.tv — warehouse, fleet
  • sales@reckord.tv — sellers, coordinators
  • finance@reckord.tv — finance team
  • crew@reckord.tv — all crew members
  • management@reckord.tv — directors, managers
  • admins@reckord.tv — platform administrators
4
Looker Studio access
Looker Studio is free at lookerstudio.google.com. Sign in with your Workspace account. No additional setup needed — you will connect it to Sheets in Phase 5.

Create the file

4
Add data validation to Enum columns
For status and enum columns, add Google Sheets data validation so users can't type invalid values directly in the sheet. Select the column → Data → Data validation → Dropdown → add allowed values.

1.2 — Connect AppSheet and create the app
Link the Sheet, let AppSheet detect the schema
▼
1
Create new AppSheet app from your Sheet
Go to appsheet.com → + New app → Start with existing data → Google Sheets. Select ReckordDB_Master from your Drive. AppSheet will auto-detect all 12 tabs and import the schema.
2
Verify detected column types
In AppSheet editor: Data → Tables → [each table] → View columns. Check that:
  • ID columns are set to Text with UNIQUEID() as initial value
  • Date columns are Date type
  • Reference columns (ClientID in Projects, etc.) are set to Ref pointing to the correct table
  • Enum columns show the correct dropdown values
Correct any misdetected types manually.
3
Configure Ref (relationship) columns
For each foreign key column, set the type to Ref:
  • Projects.ClientID → Ref to Clients, label column: CompanyName
  • Offers.ProjectID → Ref to Projects, label: ProjectName
  • OfferItems.OfferID → Ref to Offers
  • CrewAssignments.CrewID → Ref to Crew, label: Name
  • CrewAssignments.ProjectID → Ref to Projects
  • EquipmentMovements.EquipmentID → Ref to Equipment
This enables AppSheet to show dropdowns when selecting related records.
4
Enable "Referenced rows" on parent tables
On the Clients table → View columns → Add virtual column → Related Projects → type List, formula: REF_ROWS("Projects","ClientID")

Do the same for Projects → Related Offers, Projects → Related Assignments, etc. This allows a client record to show all linked projects inline.
5
Name and brand the app
App settings → Info: set name to Reckord Codex. Under Theme: set primary color to #E10B17, background dark. Upload the Reckord logo as the app icon.

1.3 — Configure roles and access control
Who sees what — role-based views in AppSheet
▼
1
Define user roles in AppSheet
Go to Security → Roles. Create the following roles:
🔧 Operations
Fleet, Equipment, Projects (read), Maintenance
💼 Sales
Clients, Contacts, Projects, Offers, OfferItems
💰 Finance
Offers (all stages), Projects (read), Clients (read)
👤 Crew
Own assignments only, Checklists, Leave requests
📊 Management
Read-all + Dashboards. No direct data edit.
⚙️ Admin
Full access. Manages users, reference data, config.
2
Assign users to roles
In AppSheet: Security → Users → Add users. Add each team member's @reckord.tv email and assign their role. You can also use Google Groups — add the group email and assign a role to the entire group.
3
Set row-level security for Crew
Crew members should only see their own assignments. On the CrewAssignments table → Security filters → add:
USEREMAIL() = [CrewEmail]

This ensures a crew member logging in only sees rows where their email matches the assigned crew email.
Phase 1 outcome — you can now:
  • Open AppSheet and see a working app connected to live Sheets data
  • Add a client record without touching the spreadsheet
  • Add a project linked to a client
  • Each team member logs in and sees only their permitted data

2.1 — Fleet management views
Vehicle records, status tracking, service alerts
▼
1
Build the Fleet list view in AppSheet
In AppSheet → Views → + New view. Create a Deck view named Fleet on the Vehicles table. Configure:
  • Title: [Name]
  • Subtitle: [RegistrationNumber] · [Type]
  • Summary column: [Status]
  • Image: Optional vehicle photo field
2
Add colour coding by status
On the Status enum column → Column settings → Valid if / Color. Set:
  • Available → Green
  • On Project → Blue
  • Maintenance → Orange
  • Out of order → Red
These colours appear as badges in the list view.
3
Add conflict prevention with a formula
On the Vehicles table → add a virtual column IsAvailableForPeriod. When a new assignment is being created, use an AppSheet expression to check for overlaps:
// Virtual column on Vehicles — checks for booking conflicts COUNTIF( SELECT( CrewAssignments[VehicleID], AND( [VehicleID] = [_THISROW].[VehicleID], [StartDate] <= [_INPUTROW].[EndDate], [EndDate] >= [_INPUTROW].[StartDate] ) ) ) = 0
Note: Full conflict prevention requires an Apps Script trigger — this is built in Phase 3 automation.
4
Maintenance alert badge
Add a virtual column ServiceAlert on Vehicles:
IF([NextServiceDate] < TODAY() + 14, "⚠️ Service due", "")

Show this in the deck view subtitle. The warning appears 14 days before service is due.

2.2 — QR code generation & scanning
Every asset gets a scannable card — actions from mobile
▼
1
Auto-generate QR codes for each asset
Add a column QRCode to the Equipment tab. Use this Apps Script function to generate and store a QR URL automatically when a new row is added:
function generateQRCodes() { const sheet = SpreadsheetApp.getActive() .getSheetByName('Equipment'); const data = sheet.getDataRange().getValues(); const qrCol = data[0].indexOf('QRCode'); const idCol = data[0].indexOf('EquipmentID'); for (let i = 1; i < data.length; i++) { if (!data[i][qrCol] && data[i][idCol]) { const id = data[i][idCol]; const qrUrl = `https://chart.googleapis.com/chart?chs=200x200&cht=qr&chl=${encodeURIComponent(id)}`; // Google Charts — no external service sheet.getRange(i + 1, qrCol + 1).setValue(qrUrl); } } }
Run this manually or set a trigger: Apps Script → Triggers → On sheet edit.
2
Display QR in AppSheet asset card
In AppSheet → Equipment table → set QRCode column type to Image. In the detail view for each asset, drag the QR code image to a prominent position. Warehouse staff can now show the phone screen to a handheld scanner, or print labels.
3
Enable scanning from AppSheet
In AppSheet: UX → Views → + New view → type Form, table: EquipmentMovements. Set the EquipmentID field to use Scan as input type.

When a warehouse user taps this view, the phone camera opens. Scanning a QR code auto-fills the asset reference and shows the asset record instantly.
4
Build fast scan actions
In AppSheet → Behavior → Actions. Create quick-action buttons that appear after a QR scan:
  • Check Out to Project — opens a form to select project, updates Status to "Assigned"
  • Check In — sets Status to "In Stock", Location to "Warehouse"
  • Assign to Truck — sets Location to selected vehicle
  • Report Damage — opens a form, creates a Maintenance record, sets Status to "Faulty"
Each action updates the relevant Equipment row and adds a row to EquipmentMovements for the audit trail.

3.2 — Offer builder with automatic pricing
Guided wizard, line items, margin calculation
▼
1
Build the offer wizard as a multi-step form
In AppSheet → Views → create a Form view on Offers. Use page break columns to split the form into steps:
  • Step 1: Select project (Ref to Projects)
  • Step 2: Offer details — validity date, internal notes
  • Step 3: Add line items (inline OfferItems table)
  • Step 4: Review totals — show calculated fields, margin, any alerts
2
Automatic pricing formulas
On the OfferItems table, add a computed column:
// TotalPrice virtual column on OfferItems [Quantity] * [UnitPrice] // TotalPrice on Offers (sum of all line items) SUMIF(OfferItems[OfferID], [OfferID], OfferItems[TotalPrice]) // DiscountedTotal on Offers [TotalPrice] * (1 - [DiscountPercent] / 100) // Margin (requires CostPrice column on OfferItems) ([TotalPrice] - SUM(OfferItems[CostPrice])) / [TotalPrice] * 100
3
Soft reservations on offer confirmation
When an offer item references a vehicle or asset, set ReservationStatus to Tentative automatically using an AppSheet action triggered on item save.

When the offer is accepted (status → Confirmed), an Apps Script trigger updates all linked reservations to Confirmed and updates asset/vehicle Status to Reserved.
4
Discount approval gate
Add a validation rule to the Offers table that triggers when DiscountPercent > 5:
// Require approval for discounts above 5% IF([DiscountPercent] > 5, [ApprovalStatus] = "Approved", TRUE)
Additionally create an AppSheet Automation (Bots section): when DiscountPercent changes to > 5, send an email notification to the manager with a link to the offer for approval. The manager approves directly in AppSheet.
5
Generate offer PDF
Create a Google Docs template for the offer PDF in 📁 Offers/Templates/. Use Apps Script to merge data:
function generateOfferPDF(offerId) { const templateId = 'YOUR_TEMPLATE_DOC_ID'; const doc = DocumentApp.openById( DriveApp.getFileById(templateId) .makeCopy(`OFF-${offerId}`).getId() ); // Replace {{placeholders}} with offer data doc.getBody().replaceText('{{OfferID}}', offerId); // ... more replacements doc.saveAndClose(); // Convert to PDF and save to Drive DriveApp.getFileById(doc.getId()) .getAs(MimeType.PDF); }
Trigger this from an AppSheet action button on the Offer detail view: "Generate PDF".

3.3 — Finance lifecycle tracking
From deposit request to invoiced and closed

Map the offer stage pipeline
Configure OfferStatus as a strict enum with ordered values. In AppSheet, use a Progress column type to display the current stage as a visual progress bar.

Stages: Draft → Sent → Negotiation → Accepted → Deposit Requested → Deposit Paid → Active → Completed → Invoiced → Closed

4.1 — Crew mobile view
What crews see when they open the app on their phone
▼
1
Create a crew-specific home view
In AppSheet → Views → create a Dashboard view visible only to the Crew role. It should contain two panels:
  • My Today — filtered CrewAssignments showing only today/upcoming assignments for the logged-in user
  • My Checklist — equipment assigned to the user for the current project
Use USEREMAIL() to filter automatically — no manual selection needed.
2
Equipment checklist with confirmation
Add a ChecklistItems table (or reuse EquipmentMovements) with a Confirmed boolean column. In the mobile view, show each item as a checkbox. When all items are confirmed, trigger an action that stamps ConfirmedBy and ConfirmedAt — acts as the crew member's digital signature for the load-out.
3
Install the AppSheet app on crew phones
Send crew members an install link: in AppSheet → Manage → Deploy → Share → copy the install URL.

On iOS: open the link in Safari → tap Share → Add to Home Screen.
On Android: Chrome will prompt to install as a PWA.

Tip: Enable AppSheet's offline mode so crew can use the checklist even without mobile data on set.
🏖️
4.2 — Leave requests & availability
Crew submits, coordinator approves, calendar updates
▼
1
Leave request form for crew
Create a Form view on CrewAvailability visible to Crew role. Fields: StartDate, EndDate, AvailabilityStatus (pre-filled to "Vacation" or "Sick"), Notes. CrewID is auto-filled from USEREMAIL().

Set initial ApprovalStatus to Pending on submission.
2
Conflict detection before approval
When a coordinator views a pending leave request, show a virtual column ConflictWarning:
// Check if crew member has project assignments in the requested leave period COUNTIF( SELECT( CrewAssignments[AssignmentID], AND( [CrewID] = [_THISROW].[CrewID], [StartDate] <= [_THISROW].[EndDate], [EndDate] >= [_THISROW].[StartDate] ) ) ) > 0
Show a red warning banner if ConflictWarning = TRUE on the approval screen.
3
Approval automation
In AppSheet → Automation → Bots → + New bot:
  • Trigger: Row added to CrewAvailability where AvailabilityStatus = Vacation
  • Action: Send email to coordinator with approve/reject links
  • When approved: set ApprovalStatus = Approved, send confirmation email to crew member
Sick leave (AvailabilityStatus = Sick) can be auto-approved — set a condition to skip the approval step.

5.1 — Prepare summary Sheets for reporting
Looker Studio reads clean aggregated data — don't expose raw tables
▼
1
Create summary tabs in the master Sheet
Add these tabs to ReckordDB_Master — these are computed tabs Looker Studio will read:
  • RPT_Pipeline — projects grouped by status with offer values
  • RPT_Utilization — vehicles and equipment utilisation %
  • RPT_Finance — offers by stage, deposit status, margin
  • RPT_Crew — crew assignments per week
  • RPT_Ops72h — projects starting in next 72h with missing resources
Use QUERY() or SUMIF() / COUNTIFS() formulas to populate these from the live data tabs — no manual updating needed.
2
Example: RPT_Utilization formula
// In RPT_Utilization, column B — vehicle utilisation this month =COUNTIFS( CrewAssignments!E:E, ">="&TEXT(DATE(YEAR(TODAY()),MONTH(TODAY()),1),"YYYY-MM-DD"), CrewAssignments!F:F, "<="&TEXT(EOMONTH(TODAY(),0),"YYYY-MM-DD") ) / COUNTA(Vehicles!A:A)
